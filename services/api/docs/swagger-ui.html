Authentication

All endpoints (except /health and public auth endpoints) require session-based authentication. Use the POST /api/auth/sign-in endpoint to authenticate and obtain session cookies.

For state-changing requests (POST, PUT, DELETE, PATCH), include the CSRF token in the x-csrf-token header. Obtain the token from GET /api/auth/csrf-token.
